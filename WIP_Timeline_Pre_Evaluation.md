## 1. 사용자의 피드백 수용 및 디자인 원칙 선언

### 🚫 [기각] 3D 홀로그램 명함 카드
*   **사용자 피드백**: *"기각. 사이트 목적성 및 디자인 미관을 해침."*
*   **수용 및 조정**: 
    - 화려한 3D 틸트와 홀로그램 셰이더 효과가 포트폴리오의 본래 목적(크리에이터의 실질적인 작품 집중)을 가리고 미니멀한 디자인 미관을 어지럽힌다는 지적에 100% 동감합니다.
    - 이에 따라 **3D 명함 카드 기획은 영구 폐기 및 기각**하며, 기존의 정갈하고 미니멀한 프로필 카드 UI(`BusinessCardContact`)를 유지하면서 가독성과 고급스러운 여백을 다듬는 미학적 미세 조정에만 집중하겠습니다.

### 💡 [보완 요구] WIP 라이브 타임라인의 구체적 기획안 수립
*   **사용자 피드백**: *"기능은 좋으나 어디에 어떻게 무슨 기능을 넣을지 구체적인 안이 필요. 무작정 넣고 보는 건 하고 싶지 않음."*
*   **수용 및 조정**:
    - 대단히 날카롭고 합리적인 제품 철학입니다. 사이트에 어설프게 새로운 피처를 끼워 넣는 대신, **정확히 어떤 위치에 어떤 방식으로 노출되고 크리에이터가 어떻게 제어하는지**에 대한 명확한 사양서(Product Specification)가 선행되어야 합니다.
    - 아래에 WIP 기능을 플랫폼에 매끄럽고 일관되게 결합할 **"구체적인 화면 설계 및 통합 사양안"**을 기획하여 보고합니다.

## 2. 구체적 위치 및 노출 방식 설계 (Where & How)

WIP 기능은 사이트의 고정 레이아웃이 아닌, 기존 BlockCanvas의 강력한 **"드래그 앤 드롭 섹션 빌더 아키텍처"** 내에 자연스럽게 녹여냅니다.

### 📍 1) 포트폴리오 섹션의 일부로 통합 (`section_type: 'wip_timeline'`)
*   크리에이터는 포트폴리오 에디터에서 기존의 '이미지 그리드 블록', '비디오 슬라이더 블록' 외에 **'WIP 라이브 타임라인 블록'**을 하나의 새로운 섹션 타입으로 생성하여 추가할 수 있습니다.
*   **자유로운 레이아웃 배치**: dnd-kit 정렬 기능이 완벽히 통합되어 있으므로, 크리에이터는 이 타임라인 블록을 포트폴리오의 **최상단**, **중간**, 혹은 **최하단(방명록 앞)** 등 자신이 원하는 최적의 미학적 위치에 드래그하여 배치할 수 있습니다.

```
+-----------------------------------------------------------+
|  [ Creator Header: Avatar & Bio ]                         |
+-----------------------------------------------------------+
|  [ Section 1: Image Grid (Selected Artworks) ]           |
+-----------------------------------------------------------+
|  [ Section 2: WIP Timeline Block (Live Workspace) ] <--- 원하는 곳에 자유 배치!
+-----------------------------------------------------------+
|  [ Section 3: Video Slider (Motion Projects) ]            |
+-----------------------------------------------------------+
```

## 3. 크리에이터 에디팅 UI 및 기능 사양 (Creator Admin Spec)

크리에이터가 포트폴리오를 관리할 때(에디터 모드) 마주할 구체적인 입력 구조 및 UI 컴포넌트 설계입니다.

### 📝 1) 타임라인 섹션 제어부
WIP 타임라인 블록 영역 우측 상단에 미니멀한 기즈모 컨트롤바가 노출됩니다.

```
+---------------------------------------------------------------------------------+
|  ● WORKSPACE LIVE LOGS                                            [+ 새 로그 작성] |
+---------------------------------------------------------------------------------+
```

*   `[+ 새 로그 작성]` 버튼을 클릭하면, 화면 중앙에 React Portal을 활용한 **글래스모피즘 입력 모달(Modal)**이 부드럽게 팝업됩니다.

### 🗂️ 2) 로그 작성 모달 사양서 (Form Spec)

```
+-----------------------------------------------------------------------------+
|  새 작업 로그 작성 (New WIP Log)                                         [X] |
+-----------------------------------------------------------------------------+
|  1. 로그 한 줄 요약 (최대 50자) * 필수                                      |
|  [ 3D Stars 배경 렌더러 프레임 레이트 모바일 60FPS 확보 성공             ] |
|                                                                             |
|  2. 작업 상세 일지 (Markdown 지원, 선택)                                    |
|  +-----------------------------------------------------------------------+  |
|  | 모바일 GPU 메모리 병목의 원인이던 canvas context 갱신 주기를 제어하여 |  |
|  | 은은한 별빛 연출을 저사양 디바이스에서도 부드럽게 구현했습니다.        |  |
|  +-----------------------------------------------------------------------+  |
|                                                                             |
|  3. 미디어 이미지 첨부 (선택)                                               |
|  +-----------------------------------------------------------------------+  |
|  |  [+] 이미지를 이곳에 드래그하여 업로드하세요 (JPG, PNG, GIF 최대 10MB)    |  |
|  +-----------------------------------------------------------------------+  |
|                                                                             |
|  4. 연관 완성작 프로젝트 링크 (선택)                                        |
|  [ 3D Stars Project v2 (Default Image Grid) ◀ 드롭다운 선택 ]                |
|                                                                             |
|                                                     [취소]  [로그 발행하기]  |
+-----------------------------------------------------------------------------+
```

*   **낙관적 업데이트(Optimistic UI) 처리**: `[로그 발행하기]` 클릭 즉시, 리스트 최상단에 0ms 만에 카드가 부드러운 페이드인 애니메이션과 함께 즉각 추가되고, 백그라운드 서버 액션이 완료되면 은은한 체크 배지가 깜빡이며 트랜잭션이 종결됩니다.

## 4. 방문자 뷰 UI 및 인터랙션 사양 (Visitor View Spec)

포트폴리오 방문자가 감상할 때의 미니멀리즘 톤 타임라인 UI 구조입니다.

```
[ WORKSPACE LIVE LOGS ]

   ●  TODAY (2026.05.23)
   │  "3D Stars 배경 렌더러 프레임 레이트 모바일 60FPS 확보 성공"
   │  
   │  모바일 GPU 메모리 병목의 원인이던 canvas context 갱신 주기를 제어하여
   │  은은한 별빛 연출을 저사양 디바이스에서도 부드럽게 구현했습니다.
   │  
   │  +---------------------------------------------------------+
   │  | [ 액자 프레임 형태의 깔끔한 가로 맞춤 스케치 캡처 이미지 ] |
   │  +---------------------------------------------------------+
   │
   │  [↗ 연관 프로젝트: 3D Stars Canvas] <--- 클릭 시, 해당 완성작 팝업 모달 스무스하게 오픈!
   │
   │
   ○  3 DAYS AGO (2026.05.20)
   │  "dnd-kit와 Lenis 스무스 스크롤 충돌 현상 우회 성공"
   │  - #dndkit #lenis #smoothscroll
   │
```

### 💫 핵심 마이크로 인터랙션 (Micro-Interactions)
1.  **스크롤 트랙 애니메이션 (Line Reveal)**:
    - 유저가 아래로 스크롤할 때, 왼쪽의 세로 점선 트랙이 스무스하게 아래로 자라나며 활성화 노드(`●`)가 빛나고, 오른쪽 로그 카드들이 `Y축 10px translate`과 함께 페이드인 렌더링을 일으킵니다.
2.  **연관 완성작 오버레이 점프**:
    - `[↗ 연관 프로젝트: 3D Stars Canvas]` 버튼에 마우스를 호버하면 해당 완성작 카드가 작게 말풍선 형태로 미니 미리보기를 제공합니다.
    - 클릭하면 별도의 페이지 이동 없이, **React Portal로 구현된 완성작 프로젝트 상세 뷰어가 스무스하게 전체 화면 위에 오버레이**되어 방문자가 "스케치부터 최종 완성작까지의 빌드 스토리"를 완벽히 감상하고 언제든지 제자리로 돌아올 수 있습니다.
