# Phase 21: 배포 전략 수립 및 환경 분리 가이드라인

## 1. 환경 분리 (Environment Separation) 개요
사이트의 안정적인 운영과 원활한 개발을 위해 개발/테스트 환경과 실제 운영 환경을 분리하는 전략을 수립함.

### 권장 환경 구성:
1. **Local (로컬):** 개발자 개인 PC 환경 (`npm run dev`)
2. **Staging (스테이징/개발 서버):** 운영 환경과 동일한 구성을 가진 테스트용 서버. 새로운 기능 및 버그 수정 사항을 운영 반영 전 최종 검수하는 용도.
3. **Production (운영 서버):** 실제 사용자가 접속하는 실 서비스 환경.

## 2. 도입 시 기대 효과
* **안정성 확보:** 운영 중인 서비스에 영향을 주지 않고 새로운 기능을 테스트할 수 있음.
* **리스크 관리:** 코드 수정 중 발생할 수 있는 에러가 실제 사용자에게 노출되는 것을 사전에 차단함.
* **데이터 보호:** 운영 DB와 테스트용 DB를 분리하여 데이터 오염 및 손실 방지.

## 3. 향후 구현 계획 (Next.js / Vercel 기준)
* **브랜치 전략:** 
  * `main` 브랜치 -> Production 배포 전용
  * `develop` 브랜치 -> Staging/Preview 배포 전용
* **CI/CD 자동화:** Git Push 시 자동으로 해당 환경에 빌드 및 배포되도록 설정.
* **환경 변수 관리:** `.env.production` 및 `.env.development`를 통해 환경별 API 키 및 DB 접속 정보 분리 관리.

## 4. 포트 구성 전략 (Port Configuration)
외부 접속을 위한 포트포워딩 및 서버 포트 할당 가이드라인.

### 기본 설정 (단일 사이트)
*   **운영 웹 포트:** 외부 `80` (HTTP) 또는 `443` (HTTPS)
*   **내부 앱 포트:** 내부 `3000` (Next.js 기본값)

### 다중 환경 구성 (운영 및 개발 사이트 분리 시)
| 환경 | 외부 포트 (공유기/방화벽) | 내부 포트 (애플리케이션) | 접속 주소 예시 |
| :--- | :--- | :--- | :--- |
| **운영 (Prod)** | 80 / 443 | 3000 | `http://mysite.com` |
| **개발 (Dev)** | 8080 (또는 별도) | 3001 | `http://mysite.com:8080` |

### 실제 공유기(iptime 등) 설정 예시
*   **서비스 포트:** `80` - `80`
*   **프로토콜:** `TCP`
*   **내부 IP 주소:** 본인 PC의 IPv4 주소 (예: `192.168.200.193`)
*   **포트 (내부):** `3000` (애플리케이션 포트)

### 대체 포트 번호 (ISP 차단 대비)
일부 통신사(ISP)에서 80번 포트를 막아둔 경우, 아래의 대체 포트를 외부(서비스) 포트로 사용함.
*   **추천:** `8080`, `8888`, `8000`
*   **주의:** 대체 포트 사용 시 접속 주소 뒤에 포트 번호를 명시해야 함 (예: `http://ip:8080`)

### 설정 시 주의사항 및 트러블슈팅
*   **내부 IP 불일치 (최우선 확인):** `ipconfig`로 확인한 본인 PC의 `IPv4 주소`(`192.168.200.138`)와 포트포워딩 설정의 `내부 IP 주소`가 반드시 일치해야 함. 공유기 주소인 `.1`을 입력하면 어떤 포트든 "공유기 사용 중" 에러가 발생할 수 있음.
*   **공유기 주소 충돌:** 공유기 자체의 관리 IP(게이트웨이) 주소로는 포트포워딩을 설정할 수 없음.
*   **관리 페이지 포트 충돌:** 일부 공유기는 `80`번 포트를 관리 페이지용으로 점유함. 이 경우 `8080` 등 대체 포트를 사용해야 함.
*   **이중 공유기 (Double NAT) 이슈:** 현재 설정 중인 공유기가 'AP 모드'인 경우 포트포워딩이 작동하지 않음. 반드시 인터넷 회선이 직접 연결된 **메인 공유기**에서 설정을 진행해야 함.
*   **유동 IP 이슈:** PC를 재부팅할 때마다 내부 IP가 바뀔 수 있으므로, 공유기 설정에서 '수동 IP 할당' 또는 'MAC 주소 기반 고정' 기능을 활용 권장.

## 5. 성공적인 설정 사례 (Case Study)
*   **서비스(외부) 포트:** `9000`
*   **내부 IP 주소:** `192.168.200.138` (본인 PC)
*   **내부(앱) 포트:** `3000` (Next.js)
*   **접속 방법:** `http://[외부IP]:9000`

## 6. Cloudflare 연동 및 보안 강화
집 서버를 도메인에 연결하고 보안을 강화하는 전략.

### DNS A 레코드 설정 (전통적 방식)
*   **설정:** 도메인 A 레코드를 외부 IP에 연결.
*   **주의:** Cloudflare Proxy(주황색 구름) 사용 시 **지원 포트 제한** 발생.
    *   지원 포트 예: `80`, `8080`, `8880`, `443`, `8443` 등.
    *   `9000`번 사용 시 **DNS Only(회색 구름)** 모드로만 사용 가능.

### Cloudflare Tunnel 도입 (권장)
*   **특징:** 포트포워딩 없이 외부 노출 가능.
*   **장점:** 외부 IP 은폐, HTTPS 자동 적용, 방화벽 설정 간소화.
*   **설정 도구:** `cloudflared` 설치 및 아웃바운드 터널 구성.

## 7. 백업 및 복구 전략
* 정기적인 DB 백업 자동화 (Supabase / Prisma 기반).
* 배포 실패 시 이전 버전으로 즉시 롤백(Rollback)할 수 있는 버전 관리 체계 강화.
