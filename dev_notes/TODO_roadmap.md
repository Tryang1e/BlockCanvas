# BlockCanvas Roadmap & Backlog (To-Do List)

이 문서는 앞으로 진행해야 할 단기 및 장기 목표, 그리고 시스템 보안 및 고도화 작업을 **기능적 구현 우선순위(Priority)** 에 따라 정리한 백로그입니다.

## 🔴 Priority 1: 핵심 기능 완성 및 치명적 보안 결함 방지 (최우선)
서비스가 정상적으로 작동하고 유저 데이터가 안전하게 보호되기 위해 **가장 먼저 코드로 구현**해야 할 작업들입니다.

* **Broken Access Control 방지 (권한 탈취 방지):**
  * 서버 액션(`Server Actions`) 및 API에서 요청자의 세션 토큰과 데이터를 대조하여 남의 게시물을 무단으로 수정/삭제할 수 없도록 백엔드 가드 구현. (시스템의 근간이므로 1순위)
* **회원가입 이메일 인증 시스템:** 
  * `is_verified` 필드 추가 및 Nodemailer (또는 Resend) 연동을 통한 이메일 인증 플로우 구현. (가짜 계정 방지)
* **[완료] 포트폴리오 공개 여부 제어:** 
  * "포트폴리오 생성/공개" 토글 기능 및 라우팅/미들웨어 권한 로직 연동.
* **[완료] 계정 관리 기능 연동:** 
  * `/dashboard/account` 내 비밀번호 변경 및 계정 탈퇴(영구 삭제) 기능 완벽 연동.
* **[완료] 에러 핸들링 및 시스템 정보 노출 방지:**
  * 서버 에러 응답 시 DB 구조나 스택 트레이스(Stack Trace)가 사용자 화면(네트워크 탭)에 노출되지 않도록 서버 액션 캐치 로직 전면 수정.
* **데이터베이스 및 쿼리 보안 (SQL Injection 방지):**
  * Prisma ORM을 사용하고 있으나, 생 쿼리가 들어간 부분이 없는지 점검하고 필터링 로직 추가.
* **[완료] 노션 스타일 에디터 UX/UI 개편:**
  * 기존의 고정된 상단 툴바를 제거하고 Tiptap의 BubbleMenu(드래그 시 메뉴 팝업) 및 FloatingMenu(빈 줄 엔터 시 블록 추가 메뉴)를 도입하여 Notion 스타일의 매끄러운(Seamless) 텍스트 작성 환경 구축.

## 🟢 Priority 1.5 : 핵심 기능 고도화 (완료)

* **[완료] 대시보드 실제 데이터 연동:** 
  * 프로젝트 뷰 카운트 API 연동 및 대시보드 누적 조회수 합산 출력 완료
  * 주간 조회수 추이 통계 기능(Mocking Data + Total Views 배분) 적용

* **[완료] Scroll 오류 개선:**
  * 스크롤 시 이미지 및 화면이 튀는 현상 등 개선 (gsap.context 적용)
  * 스크롤 시 특정 위치 고정되는 현상 개선 (스크롤 엔진 꼬임 현상 해결)

* **[완료] 크리에이터는 어드민 페이지 접속 금지:**
  * 크리에이터는 어드민 페이지 접속 시 권한 없음 발생하도록 방어 로직 추가 (middleware.ts 가드 적용)
  * 크리에이터는 어드민 페이지 접속 시 로그인 페이지로 리다이렉트 되도록 로직 추가

* **[완료] 이미지 로드 및 렌더링 최적화 롤백 복구 적용** (image.png 렌더링 에러 해결)

## 🟢 Priority 2: 프로덕션 배포 준비 및 기능 연동 (완료)
핵심 로직이 튼튼해진 후, 서비스를 대중에게 공개(런칭)하기 전에 반드시 갖춰야 할 작업들입니다.

* **[완료] 대시보드 실제 데이터 연동:** 
  * 대시보드 UI에 실제 백엔드 데이터(총 게시물, 누적 조회수 등) 연결 및 카운팅 API 구현 완료.
* **[완료] XSS 방지 (Cross-Site Scripting):**
  * `isomorphic-dompurify` 라이브러리를 도입하여 에디터 및 텍스트 렌더링 시 악성 스크립트 실행을 원천 차단 (Sanitize).
* **[완료] 모바일 화면 대응 (Responsive Design):**
  * Tailwind CSS의 반응형 유틸리티를 활용해 메인, 에디터, 대시보드의 그리드 및 레이아웃을 모바일/태블릿 친화적으로 최적화.
* **[완료] 세션 및 통신 보안 (배포 인프라 설정):**
  * **HTTPS / SSL/TLS:** 프로덕션 환경(Vercel 등)에서 자동으로 적용되도록 인프라 이관 준비 완료.
  * **쿠키 설정 강화:** 로그인 세션 쿠키에 `HttpOnly`, `Secure` (production), `SameSite=Lax` 옵션 적용.
  * **Security Headers 적용:** `next.config.ts`를 통해 `X-Content-Type-Options`, `X-Frame-Options`, `Strict-Transport-Security` 등의 보안 헤더 일괄 주입 완료.
  * **데이터 노출 방지:** 배포 시 브라우저 Source Map 비활성화.

## 🟢 Priority 3: 플랫폼 고도화 및 스펙터클 추가 (출시 후 고도화)
플랫폼이 안정적으로 운영될 때, 사용자 경험(UX)을 극대화하고 플랫폼의 가치를 높이기 위한 부가 기능들입니다.

* **AI 자동 태깅 및 카테고리 분류 (AI Integration):**
  * 이미지를 업로드하면 AI 비전 모델을 활용해 '3D', 'Blender', 'Modern' 등의 태그를 자동으로 추출 및 삽입하여 크리에이터의 수고를 덜어줌.
* **다이나믹 테마 시스템 (Dynamic Theme):**
  * 클릭 한 번으로 포트폴리오의 분위기를 완전히 전환할 수 있는 테마 시스템 (다크 모드, 글래스모피즘, 네온/사이버펑크 등).
  * CSS 변수(Variables)를 활용한 거대한 디자인 시스템 구조화.
