# Next.js Image Optimization 오류 원인 및 해결 가이드

## 에러 내용
`The requested resource isn't a valid image for /uploads/projects/1779171622731-420160047-_____2025-11-13_134231.png received null`

---

## 1. 발생 원인 분석

### 원인 A: Next.js `npm run start` (프로덕션 환경)의 런타임 업로드 제약 (가장 유력)
* **현재 상태**: 백그라운드 터미널에서 `npm run start`로 프로덕션 모드가 실행 중입니다.
* **현상**: Next.js 프로덕션 모드는 빌드 타임에 static 리소스 구조를 캐싱하고 정의합니다. 따라서 서버가 가동 중인 상태(런타임)에서 `public/uploads` 경로에 새로운 파일이 동적으로 업로드되면, Next.js 내부 Image Optimizer(이미지 최적화 API)가 이 파일을 즉시 감지하지 못하고 `404` 또는 `null`을 반환하는 고질적인 문제가 있습니다.

### 원인 B: 파일명 내 특수문자 및 공백 인코딩 문제
* **현상**: 파일명에 연속된 언더바(`_____2025-11-13_134231.png`) 또는 한글/공백이 섞여 있는 경우, 브라우저가 이미지 리소스를 요청할 때 인코딩(`%5F` 등)을 처리합니다.
* **결과**: Next.js Image Optimizer가 인코딩된 URL을 통해 파일 시스템에서 물리적 이미지 파일명을 조회하는 과정에서 서로 불일치하여 파일을 찾지 못하고 `null`을 수신하게 됩니다.

### 원인 D: 일반 `<img>` 태그를 사용한 배경화면(Banner) 역시 404가 나는 현상
* **현상**: 배경화면(Banner) 렌더링 코드(`page.tsx` 등)는 Next.js `<Image>`가 아닌 일반 HTML `<img>` 태그로 호출됩니다. 그럼에도 불구하고 엑스박스(broken image)가 뜨는 현상이 발생합니다.
* **이유**: Next.js 프로덕션 서버(`npm run start`)의 내장 정적 파일 서빙(Static File Serving) 미들웨어는 최초 빌드 시점의 `public` 폴더 구조를 기준으로 구동됩니다. 가동 중에 `public/uploads/...`에 파일이 물리적으로 성공적으로 저장되었음에도(조회 결과 실제 디바이스에 존재함), Next.js 라우팅 필터가 이를 즉각 인지하지 못하고 웹 요청에 대해 `404 Not Found`를 던지기 때문에 엑스박스로 표시됩니다.

---

## 2. 권장 해결 방안

### 해결책 1: Next.js `<Image>` 최적화 우회 (가장 확실하고 쉬운 해결 - 아바타/위젯용)
Next.js 이미지 최적화 서버를 거치지 않고 일반 정적 리소스로 직접 가져오도록 설정합니다.
* **방법 A**: Next.js `<Image>` 컴포넌트에 **`unoptimized`** 속성을 추가합니다.
  ```tsx
  <Image 
    src={imageUrl} 
    alt="Project Image" 
    width={500} 
    height={300} 
    unoptimized // <- 이미지 최적화 우회 속성 추가
  />
  ```
* **방법 B**: Next.js `<Image>` 대신 HTML 표준 **`<img>`** 태그를 사용합니다. (현재 프로젝트의 `ProjectCard.tsx` 등 대부분의 대시보드 컴포넌트는 이미 일반 `<img>` 태그를 사용하여 이 문제를 예방하고 있습니다.)

### 해결책 2: 업로드 시 고유한 영문+숫자 파일명 생성 (장기적 예방책)
업로드 시 원본 파일명을 그대로 쓰는 대신, UUID나 타임스탬프 기반의 고유한 영문/숫자 문자열로 파일 이름을 강제 리네이밍하여 저장하도록 업로드 API를 개선합니다. 한글이나 연속된 언더바 등 특수문자 파일명이 생성되지 않으므로 URL 인코딩 버그를 완전 방지할 수 있습니다.

### 해결책 3: 로컬 개발 시 `npm run dev` (개발 모드) 구동 (배경화면 엑스박스 버그 즉시 해결법 ⭐)
로컬에서 개발 및 테스트를 하실 때는 프로덕션 실행(`npm run start`) 대신 **개발 서버(`npm run dev`)**를 켜고 테스트하시는 것이 필수적입니다.
* **동작 방식**: `next dev` 모드에서는 Next.js가 `public` 폴더 하위를 매 요청마다 실시간 스캔 및 파일시스템 매핑을 갱신하므로, 런타임에 새로 추가된 배경화면이나 아바타 정적 이미지 리소스를 오류 없이 즉각 정상 서빙할 수 있습니다.
* **권장 행동**: 백그라운드 터미널에서 돌리고 있는 `npm run start`를 종료하시고, `npm run dev`로 변경하여 가동하시면 배경화면 및 프로필 이미지 엑스박스 현상이 단번에 해결됩니다.
