# 관리자(Admin) 페이지 세팅 지침

## 진행 사항
- `src/app/adminpage/layout.tsx` 파일 내부에 데스크탑용 사이드바 네비게이션을 구현합니다.
- `src/app/adminpage/page.tsx` 에 대시보드 뼈대를 구축합니다.

## 개발 권고/지침 사항
1. **레이아웃 통합성**: 관리자 도구는 일반 유저 페이지와 다른 독자적인 네비게이션 트리(Sidebar)가 필요하므로 `app/adminpage/layout.tsx`에서 이를 독립시킵니다.
2. **반응형 처리**: 사이드바는 모바일 환경에서 햄버거 메뉴를 통한 오버레이(Drawer) 형태로 변환되도록 대비해놓지만, 1차 목표에서는 데스크탑 환경 관리에 초점을 맞춥니다.
3. **권한 확인**: 페이지 접근 시 미들웨어(`middleware.ts`)나 서버 컴포넌트 상단에서 사용자가 관리자(`admin`) 권한인지 확인하는 로직을 나중에 쉽게 덧붙일 수 있도록 구조를 분리해둡니다.

## Phase 2: 실용성 중심 UI 개선 내역
- 이전 목업이었던 큼직한 위젯 통계 구조들을 덜어내고, **조회/수정이 직관적인 Data Table 컴포넌트 형태**를 도입했습니다. 
- 복잡한 차트보다는 단순 Text(숫자)와 증감폭만 노출하여 관리자의 인지 부하를 줄였습니다.
